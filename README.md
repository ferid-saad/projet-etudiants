  #   P r o j e t     t u d i a n t s      S p r i n t   4   # Projet Étudiants — Sprint 4## 🎯 Objectif généralAtteindre un niveau de qualité logicielle professionnel sur le microservice étudiant :- Mise en place d’une stratégie de test complète (unitaires, intégration, E2E, stress).- Traçabilité Jira via Xray et intégration GitHub ↔ Jira.- Ajout d’un microservice d’authentification basé sur Express, MongoDB et JWT.- Développement d’un frontend Next.js + Tailwind pour la gestion des étudiants et départements.---## 📂 Structure du dépôt````/projet-etudiants/├── api-spring-boot/        # Microservice étudiant (Spring Boot)│   ├── src/test/java/unit/         # Tests unitaires JUnit + Mockito│   ├── src/test/java/integration/  # Tests d’intégration Testcontainers│   └── pom.xml                     # Plugins JaCoCo + Gatling├── auth-service/          # Microservice Auth (Node.js / Express / MongoDB / JWT)│   ├── src/models/User.js│   ├── src/routes/auth.js│   └── app.js├── frontend/              # Frontend Next.js + Tailwind│   ├── app/etudiants/     # Liste + formulaire étudiants│   ├── app/departements/  # Liste + formulaire départements│   └── cypress/e2e/       # Tests E2E Cypress├── .github/workflows/     # CI/CD avec publication Xray└── docker-compose.yml     # Services API, DB, Auth, Frontend````---## ✅ Tâches réalisées### 1. Gestion de versions & Sprint- Création de la branche `version-4` à partir de `version-3`.- Création du **Sprint 4** dans Jira avec User Stories planifiées.### 2. Stratégie de tests- **Unitaires** : JUnit 5 + Mockito sur la logique métier (`EtudiantService`).- **Intégration** : Testcontainers avec PostgreSQL.- **E2E** : Cypress sur le frontend Next.js (`/etudiants`, `/departements`).- **Stress** : Gatling pour simuler charge concurrente sur l’API.### 3. Couverture de code- Plugin **JaCoCo** configuré avec seuil minimum de 80 %.- Build Maven échoue si couverture < 80 %.### 4. Intégration Jira- **GitHub ↔ Jira** : commits, branches et PRs liés aux tickets Jira.- **Xray** : gestion des cas de test, publication automatique des résultats CI.### 5. Microservice Authentification- Création du service `auth-service/` avec :  - Express pour l’API REST.  - MongoDB + Mongoose pour la persistance.  - bcrypt pour le hachage des mots de passe.  - JWT pour l’authentification.- Endpoints : `POST /auth/register`, `POST /auth/login`.- Ajout au `docker-compose.yml` avec conteneur MongoDB.### 6. Frontend Next.js + Tailwind- Création du projet `frontend/` avec App Router.- Pages distinctes :  - `/etudiants` : liste + formulaire CRUD étudiants.  - `/departements` : liste + formulaire CRUD départements.- Utilisation de **Server Components** pour la lecture (fetch/axios).- Utilisation de **Client Components** pour les formulaires interactifs.- Styling responsive avec Tailwind CSS.- Ajout d’une **Navbar** pour naviguer entre les pages.---## 🚀 Exécution du projet### Lancer le backend + frontend```bashdocker-compose up -dcd frontendnpm installnpm run dev````##Image Jira-sprint-4````![Board Jira](./docs/sprint4.png)````
